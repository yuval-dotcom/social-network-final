# StudyCircle - תסריט דמו להגנה

המטרה: להראות את האפליקציה כמו רשת חברתית אמיתית, ובמקביל לכסות את דרישות המרצה בלי להציג רשימת פונקציות יבשה.

## הכנה לפני הדמו

1. להריץ שרת ולקוח:

```bash
npm run dev:server:memory
npm run dev:client
```

2. לפתוח:

```text
http://localhost:5173/
```

3. להתחבר עם משתמש דמו:

```text
username: dana
password: demo123
```

להגשה עם Atlas: לפני הדמו להריץ `npm --prefix server run seed` ואז להתחבר עם אותם פרטים.

## סדר הצגה מומלץ

### 1. כניסה והרשאות

- להראות שיש מסך כניסה נפרד.
- להתחבר עם `dana / demo123`.
- להסביר: הסיסמה נבדקת בשרת, השרת מחזיר JWT, והלקוח שולח אותו בכל בקשת Ajax.

משפט קצר:

```text
משתמש לא רואה את האפליקציה לפני התחברות. אחרי login, React מציג את הפיד והשרת מוודא הרשאות בכל בקשה מוגנת.
```

### 2. הפיד הראשי

- לפתוח את `פיד`.
- להראות פוסטים קיימים עם משתמש, קבוצה, תאריך ותגיות.
- לפרסם פוסט חדש לקבוצת `Algorithms Study Lab`.
- להסביר: `User` מפרסם `Post` בתוך `Group`.

משפט קצר:

```text
הפיד לא מציג CRUD. הוא מציג חוויית משתמש רגילה של רשת חברתית, אבל מאחורי הקלעים הנתונים מגיעים משלושת המודלים.
```

### 3. פרופיל

- לפתוח `פרופיל`.
- להראות פרטי משתמש, חברים, קבוצות ופוסטים אחרונים.
- להסביר שזה מחבר בין `User`, `Group`, `Post` במסך אחד.

### 4. הפוסטים שלי

- לפתוח `הפוסטים שלי`.
- לבחור פוסט, לערוך טקסט או תגיות, לשמור.
- להסביר שהשרת אוכף שמשתמש יכול לערוך רק פוסטים שלו.

### 5. קבוצות

- לפתוח `קבוצות`.
- לבצע חיפוש לפי מילת חיפוש/קטגוריה/פרטיות.
- לבחור קבוצה ולראות חברים, בקשות ופוסטים.
- אם יש בקשה ממתינה, להראות אישור בקשה כמנהל.

### 6. ניהול ו-CRUD מלא

- לפתוח `ניהול`.
- להראות את מפת המודלים.
- לעבור על:
  - `ניהול משתמשים`: יצירה, רשימה, חיפוש, בחירה, עדכון, מחיקה.
  - `ניהול קבוצות`: יצירה, רשימה, חיפוש, בחירה, עדכון, מחיקה, הצטרפות ואישור.
  - `ניהול פוסטים`: יצירה, רשימה, חיפוש, בחירה, עדכון, מחיקה.

משפט קצר:

```text
ה-CRUD לא נמצא במסך הראשי כדי שהאפליקציה תרגיש אמיתית, אבל כל הדרישות זמינות דרך מסך ניהול.
```

### 7. צ'אט בזמן אמת

- לפתוח `צ'אט`.
- להראות חדר `general`.
- לשלוח הודעה.
- להסביר: הלקוח משתמש ב-Socket.io, השרת שומר הודעה ומפיץ אותה לחדר.

### 8. סטטיסטיקות D3

- לפתוח `סטטיסטיקות`.
- ללחוץ `טעינת גרפים`.
- להראות שני גרפים:
  - פוסטים לפי חודש.
  - פוסטים לפי קבוצה.
- להסביר שהנתונים מגיעים מהשרת והציור נעשה ב-D3 בתוך SVG.

### 9. Video ו-Canvas

- לפתוח `מדיה`.
- להראות נגן וידאו.
- לשנות שם בכרטיס וללחוץ `ציור כרטיס`.
- להסביר: `VideoPreview` מממש video, ו-`CanvasBadge` מצייר באמצעות Canvas API.

## שאלות צפויות ותשובות קצרות

**למה React Web ולא React Native?**  
זה React רגיל לדפדפן. הדרישה מאפשרת React, והפרויקט משתמש ב-Vite רק ככלי הרצה.

**איפה jQuery?**  
כל קריאות השרת נמצאות ב-`client/src/api/http.js` ומשתמשות ב-`$.ajax`.

**מהם שלושת המודלים?**  
`User`, `Group`, `Post`. מודלים תומכים כמו הודעות צ'אט לא נספרים כמודלים הרשמיים ל-CRUD.

**למה יש מצב memory?**  
זה מצב דמו מקומי למקרה ש-Atlas לא זמין בזמן פיתוח. מצב ההגשה הרשמי הוא `DATABASE_MODE=mongo`.

**איך הקומפוננטות מחולקות?**  
כל מסך מנהל state ופעולות, וחלקי UI קטנים מקבלים props ומציגים חלק אחד בלבד.
